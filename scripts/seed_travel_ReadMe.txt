用于 AUTOMATIC1111/stable-diffusion-webui 的小脚本，用于创建存在于种子之间的图像。

将让您选择两个（或更多）种子，然后创建多个图像，同时它在生成的两个噪声点之间变形。

运行良好的采样器有：Euler、LMS、Heun、DPM2 和 DDIM。（可能还有其他人，但不是 Eular。）

Batch Count 和 Batch Size 将被忽略并设置为 1。尝试使用 Euler a 生成图像将失败并打印错误。如果你真的想使用它，有一个复选框可以允许它。

什么是“种子旅行”？
要了解什么是“种子传播”，我首先必须对稳定扩散如何生成图片进行非常简单的解释。它从噪音开始，然后调整并戳它，直到它得到一个看起来与你给出的提示相匹配的图像。要获得所有这些初始噪音，您需要使用种子。种子用于生成一堆随机数，这将是您的初始噪声。不同的种子生成一组不同的随机数。

问题是种子是一个 32 位的值，大约有 40 亿个不同的可能值。这是很多，但与可能的初始噪声相比，它是无限小的。

如果我们将所有可能的噪音想象成一张地图，那么种子可以被视为该地图上的一个点。一个巨大的宇宙，只有几个可能的地方可以站立和观看。种子旅行让你做的是在这张地图上选择两个“点”，然后在它们之间旅行，每一步都是一种新的噪音变化，只有使用种子是无法达到的。

因此，在来自两个种子的噪声之间进行插值不仅看起来像视频一样酷，而且您还将获得您可能从未见过的图像。

用法
Destination seed(s): 种子旅行到从Seed。如果初始种子为空，则选择第一个目标种子作为起始种子。放在括号之间的种子将被忽略。（可能在测试时有用。）

Only use Random seeds：让您设置Number of random seeds而不是手动键入它们。Seed除非比较路径（见下文），否则将被忽略。

Steps：每个种子之间要生成的图像数量。

Loop back to Initial seed：到达终点时，生成图像以回到第一个种子。

Save results as video: 制作视频。

Frames per second：视频的fps。

Show generated images in ui：如果您生成大量步骤以使浏览器的生活更轻松，请禁用此选项。

"Hug the middle" during interpolation: 旧实验遗留下来的。使插值在开始和结束时更快一点。有时可能会产生更流畅的视频，但通常不会。
